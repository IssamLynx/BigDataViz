Requête pour exemplifier la notion de "big data":

SELECT ?film ?filmLabel ?image WHERE { ?film wdt:P31 wd:Q11424. ?film wdt:P18 ?image. SERVICE wikibase:label { bd:serviceParam wikibase:language "[AUTO_LANGUAGE],en". } }

Cette requête sélectionne tous les films (wdt:P31 wd:Q11424) et leur image associée (wdt:P18) et permet de les afficher avec leur nom en anglais et dans la langue de l'utilisateur. Cela permet d'obtenir un grand nombre de images de films différents (big data).

Requête pour exemplifier la notion de "high dimensional data":

SELECT ?film ?filmLabel ?image ?director ?producer ?actor ?writer ?release_date ?budget ?box_office_revenue WHERE { ?film wdt:P31 wd:Q11424. ?film wdt:P18 ?image. ?film wdt:P57 ?director. ?film wdt:P162 ?producer. ?film wdt:P161 ?actor. ?film wdt:P58 ?writer. ?film wdt:P577 ?release_date. ?film wdt:P2130 ?budget. ?film wdt:P2142 ?box_office_revenue. SERVICE wikibase:label { bd:serviceParam wikibase:language "[AUTO_LANGUAGE],en". } }

Cette requête sélectionne les mêmes informations que la précédente, mais ajoute également des propriétés telles que le réalisateur (wdt:P57), le producteur (wdt:P162), l'acteur (wdt:P161), le scénariste (wdt:P58), la date de sortie (wdt:P577), le budget (wdt:P2130), et les recettes du box-office (wdt:P2142) de chaque film. Cela permet d'obtenir une grande quantité d'informations sur chaque film (high dimensional data).
